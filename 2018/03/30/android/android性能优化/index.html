android性能优化

2018-03-30
android

简介:

这里简单介绍下android性能优化的一些方法

  1. 用静态工厂方法代替构造方法
    class Product{
    public static Product getInstance(){ return new Product();}
    }
    静态工厂方法可以有具体的名称
    静态方法不一定返回新的对象，可以返回已存在的对象
    静态方法不一定返回当前类的对象，也可以返回当前类的子类对象

  2. 避免创建重复的对象
    使用StringBuilder连接代替+
    使用单例模式提供常用对象

  3. 防止内存泄露

  4. 返回零长度的集合而不是null
    如果方法的返回类型是一个集合，在没有数据的情况下返回0长度的集合，这样处理方法返回值时不用处理null值，避免空指针异常。

  5. 通过接口引用对象
    List list = new ArrayList;
    这样换一种实现只需要使用新的实现该接口的类创建对象即可。

  6. 避免ANR
    避免在主线程处理耗时过长的任务，使用线程，Handler,AnsyTask, 等都可以。

  7. 执行时间测试
    通过System.currentTimeMillis()获取时间进行计算

  8. 内存消耗测试
    public class Memory{

    public static long used(){
       long total = Runtime.getRuntime().totalMemory();
       long free = Runtime.getRuntime().freeMemory();
       return (total - free);
}

    }
    通过调用used()方法测出消耗内存数

  9. 性能测试工具 traceview
    通过调用Debug.startMethodTracing(“文件名”)；
    Debug.stopMethodTracing();
    traceview 文件路径 查看文件。
